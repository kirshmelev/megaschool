# Shopping Assistant (MVP)

Обоснование проекта [тут](/article.md).
Настоящий README [тут](/README_implementation.md), ну потому что)

AI-first сервис, упрощающий выбор товаров на маркетплейсах за счёт формализации пользовательских требований, анализа предложений и выдачи объяснимых рекомендаций.
Первичный фокус — доступность и снижение когнитивной нагрузки.

## 1. Назначение проекта

MVP предназначен для проверки следующих гипотез:

* пользователи готовы делегировать выбор товара AI-ассистенту;
* мультимодальный ввод (текст / голос / изображение) упрощает формулирования запроса;
* ограниченный список объяснимых рекомендаций ценнее «бесконечной выдачи» маркетплейса;
* подход одинаково полезен как людям с ОВЗ, так и массовому пользователю.

## 2. Что входит в MVP

### Функциональный минимум

**1. Ввод запроса**

* текстовый ввод (обязательно);
* голосовой ввод (опционально, через speech-to-text);
* изображение (опционально, как уточняющий сигнал).

**2. Формализация требований**

* извлечение параметров товара (категория, цена, ключевые характеристики);
* нормализация размытых требований («нормальный», «надёжный», «не дорого»);
* выявление приоритетов (что важно, а что вторично).

**3. Поиск и агрегация**

* получение предложений с 1–4 маркетплейсов (через API или парсинг);
* унификация карточек товаров в общий формат.

**4. Оценка и ранжирование**

* базовая фильтрация по цене и параметрам;
* скоринг по:

  * Цене
  * рейтингу,
  * количеству и структуре отзывов,
  * простым признакам накрутки (аномалии распределений);

**5. Выдача результата**

* 3–5 рекомендаций;
* краткое объяснение: «почему именно этот товар»;
* возможность задать уточняющий вопрос или изменить требования.

## 3. Что не должно входить в MVP

* оформление заказа и оплата;
* аккаунты и персонализация;
* сложная антифрод-аналитика;
* масштабирование на десятки маркетплейсов.

## 4. Архитектура (упрощённо)

**Frontend**

* Web-интерфейс (mobile-first);
* минималистичная навигация;
* Практики цифровой доступносьти;
* крупные интерактивные элементы.

**Backend**

* API-сервер;
* модуль интерпретации запроса (LLM);
* модуль агрегации данных маркетплейсов;
* модуль скоринга и ранжирования.

**AI-слой**

* LLM для:

  * понимания пользовательского запроса;
  * нормализации требований;
  * генерации объяснений;
* простые эвристики для оценки отзывов.

## 5. Пример пользовательского сценария

1. Пользователь вводит:
   «Нужны беспроводные наушники, чтобы нормально слышать подкасты, не дороже 5 тысяч».
2. Сервис уточняет при необходимости:
   «Важно ли активное шумоподавление?»
3. Сервис анализирует предложения.
4. Пользователь получает 3 рекомендации с пояснениями:

   * цена,
   * сильные и слабые стороны,
   * почему выбран именно этот вариант,
   * Если вы, приемущественно собирайтесь слушать подкасты, а не музыку, то вам подойдут наушники значительно дешевле 5000 рублей.

## 6. Метрики MVP

* Кол-во использования изображений и голосовых запросов;
* время до первого полезного результата;
* количество уточнений запроса;
* доля пользователей, которым хватает первой выдачи;
* субъективная оценка «насколько стало проще выбрать».
